# PRD｜GPS 定位 + 网络信号 + 现场标定工具

- 版本：v0.1
- 日期：2026-06-28
- 范围：网页端《沙之书》（候鸟300）实景地图的 GPS 定位基础设施——实时定位采集、网络/GPS 信号 HUD、经纬度↔地图坐标的标定换算层，以及供现场操作员使用的可视化标定工具（结果服务器共享）

## 1. 背景与目标

沙城地图由真实场馆（鸟其林内场）建模而来，因此存在一条固定的「真实经纬度 ↔ 地图坐标」映射关系。本期目标是把这条映射做成可用的基础设施：

- 网页端读取玩家真实 GPS 定位（浏览器原生 `geolocation`，HTTPS 即可用）。
- 提供经纬度 → 航拍源坐标 → 游戏瓦片坐标的换算层，平面场地用二维仿射变换标定。
- 现场操作员可用可视化工具采集锚点、当场解算、验收精度并保存；标定结果存服务器，全场玩家共享同一套变换。
- 同时给出 GPS / 网络信号的实时状态 HUD。

不在本期：GPS 驱动角色移动（标定链路已就绪，留作下一环）、扫码签到点兜底、室内信标/UWB 定位。

## 2. 关键结论与取舍

- **可行性**：拿定位（浏览器 API）零难度；难点在「经纬度→像素」的标定，以及室内精度。
- **精度瓶颈**：室外开阔 GPS 约 ±5~10m，**室内/内场可能 ±20~50m**，是落地的真正约束。本期先把标定链路打通并可视化验收，精度问题在数据采集阶段用「多锚点 + 最小二乘平均 + 残差校验」缓解，长期建议叠加扫码签到点兜底。
- **标定存储**：选 Convex 服务器单例而非 localStorage——一名操作员标定一次，全场共享，不让每台设备各自重标（DRY / 单一真相源）。

## 3. 坐标链路

```
真实经纬度 (lat,lng)
  └─ 二维仿射变换（最小二乘，≥3 锚点）→ 航拍源坐标 source(x,y)  [1703 × 1279]
       └─ 线性缩放（source/SOURCE × worldMap.size）→ 瓦片坐标 tile(x,y)
```

- 平面场地用仿射变换（旋转 + 缩放 + 错切 + 平移）即可；几百米尺度内把经纬度当平面坐标，墨卡托畸变可忽略。
- 瓦片坐标即玩家 `player.position` 所用坐标系，与现有 `moveTo` 移动系统天然对接。

## 4. 详细设计

### 4.1 换算与标定核心 `src/lib/gps.ts`

纯函数层，可单测：

- `fitAffine(anchors)`：从 ≥3 个互不共线锚点用最小二乘（normal equations + 3×3 高斯消元）解仿射系数；共线/不足返回 `null`。
- `gpsToSource` / `sourceToTile` / `gpsToTile`：三段换算，未标定返回 `null`。
- `haversineMeters`：两点地表距离（米），用于精度展示与测距。
- `makeCalibration` / `isCalibrated`：构造与判定。
- `VENUE_GPS_ANCHORS`（baked 占位，TODO 待现场实测）+ `VENUE_CALIBRATION` 作为服务器无数据时的回退。

### 4.2 定位采集 `src/hooks/useGeolocation.ts`

- 封装 `navigator.geolocation.watchPosition`（`enableHighAccuracy`），**按需授权**（`enabled=false` 不请求权限，避免一进页面就弹框）。
- 状态机 `GeoStatus`：`idle / unsupported / locating / active / denied / error`。
- 输出经纬度、精度（米）、朝向、速度、时间戳；`gpsSignalBars(accuracy)` 把精度映射成 0~3 格信号。

### 4.3 网络状态 `src/hooks/useNetworkStatus.ts`

- `navigator.onLine` + `navigator.connection`（`effectiveType` / `downlink` / `rtt`），监听 online/offline/change。
- `networkSignalBars(net)`：离线=0，按 effectiveType（4g/3g/2g）或 rtt 回退映射 0~3 格。

### 4.4 信号 HUD `src/components/SignalHud.tsx`

- 屏幕左上角轻量状态条：GPS（点击才开启定位授权）+ 网络两段，各带 3 格信号与读数（`±xx m` / `4G` / `离线`）。
- 展开看详情：经纬度、精度、网络 RTT、**地图标定状态**（读服务器实时标定）。
- 暖学术棕色系，内联 SVG 信号条，无 emoji。

### 4.5 标定数据 `src/hooks/useCalibration.ts` + 后端

- `convex/schema.ts` 新增 `gpsCalibration` 单例表：`{ anchors: [{lat,lng,sourceX,sourceY,label?}], updatedAt }`。
- `convex/gpsCalibration.ts`：`getCalibration`（取最新一条，无则 `[]`）/ `saveCalibration`（覆盖单例 upsert）。
- `useCalibration()`：优先服务器锚点，无则回退 baked 占位；`useMemo` 实时解算；暴露 `saveAnchors`。

### 4.6 现场标定工具 `src/components/CalibrationPanel.tsx`

操作员专用，由设置·调试区开关挂载（见 4.8）。

- **采集流程**：站到地标点 →「采集 GPS」（抓当前定位）→ 在地图上点对应位置 → 合成一组锚点；重复 ≥3 组（要求互不共线、散开成三角形）。
- 地图点击通过 4.7 的采集总线换算成源坐标，标定态下不移动角色。
- **实时反馈**：锚点列表（可删）、解算状态（`N 个锚点 · 还差 M / 可解算`）、**重投影残差（px，越小越准）**。
- **预览定位**：用当前真实 GPS 反推地图坐标、在地图上打 ping（复用 `focusMapVenue`）——当场验收精度的最快方式。
- **保存到服务器**：写入 `gpsCalibration`，全场玩家与 HUD 立即生效。
- 视觉为不透明「测绘仪表卡」：实底渐变背景（杜绝地图透出导致看不清）、LED 状态灯按精度变色、赤土色顶脊 + 四角定位刻线、状态文案全中文。

### 4.7 地图点击采集总线 `src/lib/mapFocus.ts` + `src/components/PixiGame.tsx`

- `mapFocus.ts` 新增 `setMapTapCaptureHandler / isMapTapCaptureActive / captureMapTap`，跨 Pixi↔React 渲染边界传递（沿用现有 mapFocus 总线模式）。
- `PixiGame.onMapPointerUp`：标定态优先，把点击点（瓦片）换算成航拍源坐标交给面板，`return` 不再走移动逻辑；该分支在 `humanPlayerId` 守卫之前，操作员无需先加入世界。

### 4.8 开关入口 `src/components/SettingsMenu.tsx` + `src/App.tsx`

- 标定开关放进设置 → **调试区**（`SHOW_DEV_TOOLS` 下可见），配 `LocateIcon`（`DeckIcons.tsx`）。
- `calibrating` 提升为 `App` 受控 state，沿 `showCollisionOverlay` 同一路径 `App → Game → SettingsMenu` 下传；废弃早期的 `?calibrate=1` URL 入口（单一真源）。

## 5. 使用说明（操作员）

1. 设置 → 调试 → 打开「GPS 标定」；面板出现在屏幕左上角（避开右侧手卷）。
2. 允许浏览器定位；等顶部出现 `±xx m`，「采集 GPS」按钮变亮。
3. 走到地标点 →「采集 GPS」→ 点地图对应位置 → 成一组锚点；换点重复 ≥3 次。
4. 看残差（越小越准）；「预览定位」当场验收；满意后「保存到服务器」。

## 6. 验收标准

- 网页端能读真实经纬度并在 HUD 显示精度与网络信号。
- ≥3 个互不共线锚点能解算出仿射变换并显示残差。
- 「预览定位」的 ping 落点与操作员实际所在位置一致（室外精度内）。
- 保存后刷新/换设备，HUD 显示「已标定」，标定对全场生效。

## 7. 风险与后续

- **室内精度**：内场 GPS 可能漂移 ±20~50m，采集时需等精度数字稳定再采，单点漂移过大则删除重采；长期叠加扫码签到点兜底。
- **生产可见性**：调试区受 `SHOW_DEV_TOOLS = DEV || VITE_SHOW_DEBUG_UI` 控制，现场部署版需设 `VITE_SHOW_DEBUG_UI=1` 才能露出标定开关。
- **下一环（未做）**：GPS 驱动角色移动——加「GPS 模式」开关，标定后持续把真实定位 `moveTo` 到对应瓦片；需在标定精度验收通过后再接。
